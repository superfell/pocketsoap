DocGen

(a.k.a. "IDL + XML = HTML + Happiness")

by Shawn VanNess

 

Description:

The DocGen Wizard allows you to generate reference docs for your object models -- it "weaves" together API documentation (provided in an XML file) with API elements defined in the IDL file, to produce a folder full of HTML files.

In addition to raw HTML files, the DocGen Wizard also produces the table-of-contents, index, and project files needed by the Microsoft HTML Help Workshop compiler.  All that is left to do is run hhc.exe!

If documentation is missing for one or more API elements, a warning message is displayed, and placeholder text is used in its stead.  The DocGen Wizard also provides an option to automatically "stub-out" the XML file with placeholder text for the missing docs.

The DocGen Wizard will really come into its own when adapted for the next release of Gen<X>, which supports the notion of "faceless exemplars".  Wizards like DocGen will become first-class build tools, which can be run from the command-line, and, therefore, be integrated into automated build processes!

Notes:

  • This version of DocGen requires Gen<X> v1.0, Windows Script 5.0, and MSXML v1.0.
  • The next release of Gen<X> will support true "silent" operation, so wizards like DocGen can be incorporated into automated build-scripts.  Such wizards are being called "faceless exemplars".

Usage:

DocGen is a stand-alone wizard, which means that it can be run from within the Gen<X> Wizard Manager or from the command line. To run it from within the Wizard Manager, double-click on DocGenWizard.gxw and press the "Run Wizard" button on the toolbar. To run DocGen from the command line, use the wizrun command line tool, e.g. 
    WizRun.exe DocGenWizard.gxw

Usage (Advanced):

To create an XML "skeleton", for a brand new IDL file, simply check the "auto-generate placeholder text" checkbox.  If the specified .xml file does not exist, it will be created for you.

Known Limitations and Bugs:

  • If DocGen finds a previously-unencountered interface (or a hidden interface) referenced inside of a coclass definition, then a scripting error will occur.
